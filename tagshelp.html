1.2. Introduction

XBS MetaTags adds Search Engine Optimisation (SEO) facilities to your Xoops web site, in particular it adds the ability to manage html metatag keyword and descriptions on a page by page basis.

Features:

- Keyword manipulation on an automatic or manual page by page basis.

- Description on a page by page basis

- Automatic setup of page parameters

- Keyword blacklisting

- Keyword whitelisting

- Keyword tracking

- Simple config file to enable your own modules to include themselves in the MetaTags schema

2.1. Prerequisites

Xoops V 2.0.13, 2.0.14 or V 2.2.4 (some limitations in admin menu appearance exist under V2.2.4 but the module is usable.)

2.2. Effecting the installation

Install XBS MetaTags as per any other module. That is, unpack the archive and copy the contents of the xbstags directory into your Xoops site modules directory. Logon as administrator, go to modules admin and install the module as per normal.

During installation, a message is sent to Xoobs.net telling the author that you have installed a copy of the module. No personal information is sent and this information is not used for any other purpose than to determine coverage of the module. If you deinstall the module a similar message is sent notifying the author of the deinstallation.

You now need to make a small change to a Xoops core file - footer.php. Where the change is, is dependent on which version of Xoops you are running:

3. Using XBS MetaTags

Using XBS MetaTags Is relatively straightforward. There are no user side screens to teh module. Every thing is done behind the scenes in the Module Administration section.

To start using it go to the Metatags administration pages and select 'Update'. Choose the modules that you want to include and hit the button. You will go to the page index list. Edit each one to set page title, description and enter a set of keywords if you wish. To see these keyords in your page set the method to 'db', otherwise use one of the auto methods to generate keywords on the fly. Please note that you may get some duplicate pages in the page index. This is because some modules use the same page for a variety of user side menu options. Delete any duplicates.

The module configuration (preferences) allows you to set default behaviour for the module.

To turn off MetaTags, simply deactivate it from the main Modules Administration screen. There is no need to remove your hack from footer.php.

3.1. MetaTags in detail

3.1.1. Module Configuration

Select 'Module Config' from teh topline menu on teh administration screen (preferences - metatags on Xoops 2.2.4). You can set the following system wide options.

Page Title - Decide to use teh Xoops page title as set in teh Xoops Preferences - General admin page or use teh MetaTags page title. Using MetaTags allows you to have a page by page title displayed in teh title bar of teh browser.

Page Description - Like teh title, choose teh Xoops title or a MetaTags (page by page) title.

Keyword Generation method - The default setting for new pages added to MetaTags. Choose:

  1. Use database - Use manually entered keywords for each page

  2. Auto - Text order - generate keywords for teh page in the order in which they appear on teh page

  3. Auto - Least frequent first - generate keywords for teh page basd on teh page content but order them with teh least frequently occuring words first in teh list

  4. Auto - Most frequent first - generate keywords for teh page basd on teh page content but order them with teh most frequently occuring words first in the list

  5. Use Xoops Default - Use the keywords that were entered into the Xoops general preferences admin screen

Maximum keywords - Maximum number of keywords to generate. This is the default value used by new pages added to MetaTags

Min. keyword length - the minimum length of a word to be added into the keywords list.

Track Keywords - Set yes or No. Switch this one to track keywords being generated by each page so that you can then blacklist inappropriate words.

Max Track pages - The number of times each page will be tracked to create a keywords list. Keyword tracking does take a small amount of time so don't set this figure too high.

3.1.2. Page Index

This screen lists all the pages that you are manipulating with MetaTags and allows you to edit their parameters.

You may also manually add a page to the list. You may wish to do this for a module page that does not get automatically included by the Update process. If for instance, a page is displayed by a module that is not as a result of a user menu selection (perhaps as a result of pressing a button or clicking on a link within a page), the Update process will not find it and you will have to manually add it.

You can delete pages from the list. Quite often, module developers, will use a single page to process the users menu selections and then pass control to another page for the actual processing. If, as a result, of the Update process, you find you have duplicate page entries, delete as many as are required to reduce the number of duplicates to 1 (i.e. a single listing)

3.1.2.1. Editing a page entry

By clicking on the edit or insert link from the page index, you will go to the page details edit screen. Enter or amend the following details:

Page Title - A title for the page

Page Description - A description for the page

Keywords - Enter a list of keywords separated by commas. If you set the Key Method to 'db', this list of words will be used. As an aid, there is a scroll box of keywords collected by the page tracking mechanism for the page (as long as a/ you have switched page tracking on in module config and b/ the page has been hit enough times to generate a list of words.) Select a word and click the insert button to add it to the list

Keyword Method - Choose the keyword generation method for this page

  1. db - Use manually entered keywords

  2. textorder - generate keywords for the page in the order in which they appear on the page

  3. leastorder - generate keywords for the page based on the page content but order them with the least frequently occurring words first in the list

  4. mostorder - generate keywords for the page based on the page content but order them with the most frequently occurring words first in the list

  5. xoops - Use the keywords that were entered into the Xoops general preferences admin screen

Max key words - Maximum number of keywords to generate.

Min. key length - the minimum length of a word to be added into the keywords list.

3.1.3. Update

A list of modules that do not have pages listed in the Page Index is presented here. Select the modules that you wish to include (Ctrl + space or Ctrl + mouse click to select multiple entries.) Click the submit button.

3.1.4. Blacklist

The Blacklist is a list of words that are never included in a MetaTags generated keyword list. You can manually enter them separated by commas or insert them from list of words displayed on the screen (you can multi select words). You can also enter regular expressions enclosed by / /.

The list of words is generated by the page tracking mechanism that is switched on in the module configuration. You need to have had this running for a while to generate a comprehensive list. Please also note that you may to submit this screen a number of times as the list will change as newly entered blacklist words are removed from it.

3.1.5. Whitelist

The Whitelist is a list of words that you want including in every keyword list generated by MetaTags. Simply enter them separated by commas.

3.1.6. Tracking Control

Here you can see which pages have been tracked and how many times they have had their word list stored in the database. If you substantially change a page, you might want to delete the page track so that page tracking starts collecting again.

3.1.7. Preparing your own module for MetaTags

If you have developed your own module and you want to make it easy for the user to include it in their MetaTags setup, simply create a metatags_info.php file and place it in teh root of your module directory. See the metatags_info.php file included with this module for more details.
